# Трекер задач
Учебный проект системного аналитика: трекер задач для команды

## Цель
Трекер должен закрывать потребности команды в контроле выполнения рабочих процессов, а так же предусматривать контроли доступа, работу с документами, иерархию задач.
Трекер не должен пытаться дублировать мессенджеры и видеочаты.

### Необходимо достичь:
- Уменьшения количества просроченных и невыполненных задач <br>
- Отстутствия совещаний вида "кто чем занимается и чего достиг?" <br>

## Пользователи
- Руководитель команды - ответственнен за распределение задач, пролей и прав между командой, ставит задачи, контролирует срок их выполнения, оценивает результат <br>
- Участник команды - получает задачи на исполнение и выполняет их, может иметь права доступа на обсуждение и оценку других задач, постановку определенных задач<br>
